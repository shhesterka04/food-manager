Идея: создатьоднопользовательскоедесктопное приложение, которое умеет подбирать оптимальный рацион питания на неделю на основе полученных данных из опроса пользователя. Зная рост, пол, вес и т.д., программа вычисляет по формулам необходимое количество калорий, белков, жиров и углеводов.
В программе будет база данных с названиями блюд и их параметрами (Ккал, БЖУ), на основании которой будут подбираться оптимальные рационыс помощью алгоритмов динамического программирования. Все это будет оформлено в графическом интерфейсе: стартовый тест исам конструктор. Программа должна помнить данные, которые были введены в стартовом тесте, должна присутствовать возможность стереть данные и пройти тест заново.
Каждый раз рацион должен генерироваться новый: не должно быть 1 наиболее оптимального рациона, который будет предлагаться каждый раз.
У обычных пользователей нет права создания нового рецепта, но он может их предлагать. Они могут только создавать рационы и сохранять их.
У администраторов есть права обычных пользователей + проверка предложенных рецептов (отказать, принять), создание рецептов.
Личные задачи:
-Научиться делать приложения с графическим интерфейсом
-Научиться «связывать» C++ и Базы данных
-Сделать алгоритм, который умеет оптимально комбинировать блюда, примерно придерживаясь поставленных рамок по калориям
Стек технологий:
-SQLite
-Qt
