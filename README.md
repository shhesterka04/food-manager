**Идея:** создать многопользовательское десктопное приложение, которое умеет подбирать оптимальный рацион питания на неделю на основе полученных данных из опроса пользователя.
Во время опроса программа узнает следующие характеристики пользователя:
- Рост
- Вес
- Пол
- Возраст
- Образ жизни
- Цели по питанию
Далее вычисляется оптимальный уровень калорий, белков, жиров и углеводов при помощи идей динамического программирования.
Конечная работа будет оформлена в графический интерфейс Qt. Какие страницы будут в приложени:
1) Главное меню (рацион на неделю, рецепт, кнопки на профиль и рецепты)
2) Профиль (инфа о себе)
3) Рецепты (список рецептов с сортировкой, фильтрами и тегами)
Программа должна помнить данные, которые были введены в стартовом тесте, должна присутствовать возможность стереть данные и пройти тест заново. Каждый раз рацион должен генерироваться новый: не должно быть 1 наиболее оптимального рациона, который будет предлагаться каждый раз.
**Уровни пользователей:**
У обычных пользователей нет права создания нового рецепта, но он может их предлагать. Они могут только создавать рационы и сохранять их.
У администраторов есть права обычных пользователей + проверка предложенных рецептов (отказать, принять), создание рецептов.
**Личные задачи:**
- Научиться делать приложения с графическим интерфейсом 
- Научиться «связывать» C++ и Базы данных 
- Сделать алгоритм, который умеет оптимально комбинировать блюда, примерно придерживаясь поставленных рамок по калориям
**Стек технологий:** 
- SQLite 
- Qt 
**Для кого:** 
- Люди, которые хотят нормально питаться, но не хотят много думать над вопросом «А что поесть?», но денег не так много, чтобы брать готовую еду: студенты, жители регионов


1) Написать калькулятор калорий
2) Написать опросник в консоли для курсача
3) Написать алгоритм составляения рациона питания
4) Составить тестовый датасет для тестирования
5) Сделать главную страницу приложения
6) Сделать страницу "профиль"
7) Сделать страницу "рецепты"
